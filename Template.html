Templates(de funções)



Templates são uma alternativa à sobrecarga de funções, quando envolve lógica de programação idênticas para varios tipos de entrada de dados.

Quando você cria uma função normal em C++, ela é específica para um tipo de dado. Por exemplo, uma função que soma dois números inteiros não funcionará diretamente para números decimais. Aqui é onde entram os templates de funções, e até mesmo templates de classe que veremos no decorrer desta página.

Com templates de funções, você pode criar uma função que funciona para diferentes tipos de dados. Por exemplo: 


                        
                        #include <iostream>
                        using namespace std;
                                
                        template<typename   NewType>
                        void  imprimi( NewType numero1,  NewType numero2,  NewType numero3,  NewType numero4);
                                
                                
                        template<typename   NewType>
                        void  imprimi( NewType numero1,  NewType numero2,  NewType numero3);
                        
                                
                        template<typename   NewType> 
                        void  imprimi( NewType numero1,  NewType numero2);
                                
                                
                        template<typename   NewType>
                        void  imprimi( NewType numero1);
                                
                        int main()
                        {
                            cout  <<  "Hello REAA!"  <<  endl;
                             imprimi(99, 44, 3, 7);
                             imprimi(1, 8, 37);
                             imprimi(17, 66);
                             imprimi(1025);
                                
                                
                                
                                
                            return 0;
                        }
                                
                                
                                
                        template<typename   NewType>
                        void  imprimi( NewType numero1,  NewType numero2,  NewType numero3,  NewType numero4)
                        {
                            cout << "imprime: " << numero1 << '-' << numero2 << '-' << numero3 << '-' << numero4 << endl;
                        }
                                
                                
                        template<typename   NewType>
                        void  imprimi( NewType numero1,  NewType numero2,  NewType numero3)
                        {
                            cout << "imprime: " << numero1 << '-' << numero2 << '-' << numero3 << endl;
                        }
                                
                                
                        template<typename   NewType>  
                        void  imprimi( NewType numero1,  NewType numero2)
                        {
                            cout << "imprime: " << numero1 << '-' << numero2 << endl;
                        }
                                
                                
                        template<typename   NewType> 
                        void  imprimi( NewType numero1)
                        {
                            cout << "imprime: " << numero1 << endl;
                        }

Veja a sintaxe da construção de um template de funções, antes do protótipo da função e até mesmo antes da declaração do desenvolvimento da dela, é declarado a palavra reservada template onde, template indica que a função será um template, typename será um corringa para um tipo de dado, isso significa que o usuário irá colocar algum tipo de dados (int, double, string.. ) e o compilador vai identificar qual tipo de dado e substituir o typename em tempo de compilação. A variável NewType é um argumento qualquer e eu poderia ter colocar qualquer coisa.

Template não tem muito mistério, mas é importante fazer leitura de algumas obras para te conhecimento sobre boas práticas e dicas de uso em projetos reais.
