Orientação a Objetos é um modelo de programação que visa modelar algo do mundo real encapsulada em uma unica unidade chamada classe que ao instânciar no corpo main, estará criando um objeto. E como, todo objeto tem, caracteristicas, identidade e comportamento, a maneira como este objeto se relaciona com os outros é o que faz nascer o conceito OOP, um objeto orientado a fazer algo.

Este, sem dúvida, é um tema altamente relevante na linguagem C++, na minha opinião. Para aqueles familiarizados com programação em C, é possível identificar semelhanças com o conceito de estruturas de dados. Em C, frequentemente utilizamos structs para representar modelos de objetos, definidos por atributos ou tipos de dados específicos. Abaixo, apresento um exemplo que serve como lembrete sobre como é a construção e utilização de uma struct em C. Essas noções são fundamentais ao se transicionar da programação procedural para a orientada a objetos em C++. 

                    
                #include <iostream>
                using namespace std;
                
                            
                //========================================
                //--definição global da struct--
                typedef struct
                {
                    int paginas;
                    double peso;
                    char *autor;
                }Livro;  //Defino novo tipo(classe) do tipo Livro 
                
                //========================================
                //--Corpo do programa--
                int main()
                {
                    printf("Hello REAA\n");
                    Livro deEleronica;                   //defino meu novo dado chamado 
                                                         //"deEletronica" do tipo estrutura livro;
                            
                    deEleronica.paginas=20;              //acesso osatributos da estrutura através do operador ponto
                    deEleronica.peso=.2500;              //acesso osatributos da estrutura através do operador ponto
                    deEleronica.autor="Lucas Alves";     //acesso osatributos da estrutura através do operador ponto
                            
                    printf("paginas: %d \n", deEleronica.paginas);  //Imprimo os dados de cada atributo
                    printf("peso: %.3f\n", deEleronica.peso);       //Imprimo os dados de cada atributo
                    printf("autor: %s\n", deEleronica.autor);       //Imprimo os dados de cada atributo
                            
                    return 0;
                }//Fim da man

Certamente! A orientação a objetos é, de maneira geral, um modelo de programação (evitei usar a palavra-paradigma) que oferece um modelo estruturado para resolver problemas. Em termos simples, ela representa uma abordagem para modelar conceitos do mundo real, utilizando a noção de objetos. No exemplo apresentado anteriormente, ilustrei essa abordagem em um pequeno programa em C, que representa a estrutura de um livro, focalizando características específicas, tais como a quantidade de páginas, o peso e o autor. Embora existam mais atributos possíveis, optei por citar apenas esses para fins didáticos. Essa simplificação ajuda a destacar a essência da orientação a objetos na prática, mostrando como ela pode ser aplicada de maneira concisa e eficaz na representação de entidades do mundo real.

No corpo do meu programa principal, eu criei uma variável chamada deEletronica do tipo estrutura Livro, eu poderia criar outra variável chamada deFisica, deGeometria, deAutoAjuda e etc... com as mesmas caracteristicas da estrutura Livro e com isso criando uma estrutura de dados. Ou uma estante de livros.

Na linguagem C++, em vez de criarmos uma struct, criamos uma classe. Dentro dela, não só utilizamos tipos de dados primitivos como int, float, double e char, chamados de membros de dados, para definir as características do objeto. Além disso, incluímos funções, chamadas de funções membros, que estabelecem comportamentos específicos. Isso faz com que uma variável, por exemplo, chamada de "deEletrônica", não seja apenas uma variável, mas um objeto. Isso acontece porque, além de possuir características, ela também tem comportamento devido às funções membros que executam tarefas dentro da classe. E ai, que que achou?

A baixo cito o mesmo exemplo da estrutura Livro. Porém, modificado conforme explicado anteriromente, em vez de struct passa a ser class e além de tipos primitivos declarados como private, temos também funções membros declarados como public. 

                    

                        
#include <iostream>
using namespace std;
#include <string>
using std::string;

                            
//========================================
//--definição global da struct--



class Livro
{
public:
    Livro(int=0, double=0.0, string=" ");                       
    void    setNumberPage(int);
    int     getNumberPage();

    void    setWeight(double);
    double  getWeight();

    void setAutor(string);
    string getAutor();

    void imprime();
private:
    int     _page;
    double  _weight;
    string  _autor;

};



//========================================
//--Corpo do programa--


int main()
{
    printf("Hello REAA\n");
    Livro deEletronica;    //Crio um objeto "deEletronica" da classe Livro

    deEletronica.setAutor("Lucas Alves");  //chamo a função membro "setAutor" para setar o nome do autor
    deEletronica.setWeight(0.4);           //chamo a função membro "setWeight" para setar o peso do livro
    deEletronica.setNumberPage(10);        //chamo a função membro "setNumberPage" para setar o numero de paginas do livo

    deEletronica.imprime();                //chamo a função mebro "imprime" para imprimir os valore dos membros de dados
                                           //do objeto criado no cropo da função man.
    return 0;
}//Fim da man

    

//========================================
//--Construtor--



Livro::Livro(int paginas, double peso, string autor)
{
    _autor=autor;
    _page=paginas;
    _weight=peso;
}//Fim do construtor
//========================================
//--Seta o numero de paginas--
void Livro::setNumberPage(int page)
{
    _page=page;
}//Fim  setNumberPage
//========================================
//--Entrega o número de paginas--
int Livro::getNumberPage()
{
    return _page;
}//Fim do getNumberPage
//========================================
//--Seta o peso do livro--
void Livro::setWeight(double peso)
{
    _weight=peso;
}//FIm do getWeight
//========================================
//--Entrega o peso do livro--
double Livro::getWeight()
{
    return _weight;
}//Fim
//========================================
//--Seta o Autor do livro--
void Livro::setAutor(string autor)
{
    _autor=autor;
}//Fim do setAutor
//========================================
//--Entrega o autor do livro--
string Livro::getAutor()
{
    return _autor;
}//Fim do getAutor
//========================================
//--Imprime na tela--
void Livro::imprime()
{
    cout<<"Numero paginas :"<<getNumberPage()<<endl;
    cout<<"peso  do livro :"<<getWeight()<<endl;
    cout<<"Autor do livro :"<<getAutor()<<endl;
}
