O polimorfismo é um dos quatro pilares da programação orientada a objetos (POO), juntamente com encapsulamento, herança e abstração. Polimorfismo, em termos gerais, refere-se à capacidade de um objeto assumir várias formas ou comportar-se de maneiras diferentes em diferentes contextos.

Em resumo, polimorfismo na programação POO se refere a função membro da classe base que pode assumir diferentes formas dentro do objeto que a invoca. 10 O que é polimorfismo em C++? (Invocação):

O polimorfismo em C++, é usado em conjunto com herança e é feito com uso de funções virtuais. Emquanto herança permite herdar as mesmas caracteristicas da classe báse. O polimorfismo, permite mudar a forma da caracteristica que está herdando.

Por exemplo!Se a classe básica animal tem uma função membro "faz barulho" e herdando na minha classe derivada gato, então gato vai fazer aquele barulho expecífico que está herdando.

Porém! Com polimorfismo, além de herdar a função membro faz barulho, posso sobreescrever com uma nova forma de fazer barulho. Posso criar minha classe derivada Automóvel, e o barulho herdado da classe animal fazer barulho de carro.

8 O que são funções virtuais em C++?

Em C++, funções virtuais desempenham um papel fundamental na implementação de polimorfismo. O polimorfismo é uma característica da programação orientada a objetos que permite que objetos de diferentes classes sejam tratados de maneira uniforme, proporcionando flexibilidade e extensibilidade ao código. Uma função virtual é uma função declarada em uma classe base que pode ser redefinida em uma classe derivada. Isso significa que, quando uma classe derivada herda de uma classe base e redefine uma função virtual, a versão da função na classe derivada será chamada quando a função for invocada através de um ponteiro ou referência da classe base.

7 O que é uma função virtual pura?

Em algumas situações, é desejável garantir que todas as classes derivadas de uma classe base tenham a implementação de um método específico. No entanto, em algumas circunstâncias, a classe base pode não ter informações suficientes para fornecer uma definição para uma função virtual. Para lidar com esses casos, C++ oferece o conceito de funções virtuais puras.

Uma função virtual pura é uma função virtual que não possui implementação dentro da classe base. Para declarar uma função virtual pura, utiliza-se a seguinte forma:

virtual tipo nome_funcao(lista_args) = 0;

6 Como funciona a ligação dinâmica em C++?

A ligação dinâmica (dynamic binding) em C++ está associada ao conceito de polimorfismo, especialmente ao polimorfismo de tempo de execução. A ligação dinâmica permite que o código invoque métodos de uma classe base por meio de um ponteiro ou referência da classe base, e a decisão sobre qual implementação específica chamar é tomada em tempo de execução.

A ligação dinâmica em C++ é alcançada principalmente através do uso de funções virtuais e ponteiros para classes base. Aqui estão os principais componentes que desempenham um papel na ligação dinâmica:

Funções Virtuais: Uma classe base pode declarar funções virtuais. Essas funções virtuais podem ser redefinidas em classes derivadas.

Ponteiro para Classe Base: Um ponteiro ou referência de uma classe base pode ser usado para apontar para objetos de classes derivadas.

Palavra-Chave virtual: A palavra-chave virtual é usada para indicar que uma função em uma classe base é virtual e pode ser redefinida em classes derivadas.

Uso de override: A palavra-chave override é usada nas classes derivadas para indicar explicitamente que uma função está redefinindo uma função virtual da classe base.

Chamada Dinâmica: Quando uma função virtual é chamada através de um ponteiro ou referência de classe base, a implementação específica da classe derivada é chamada em tempo de execução.

4 Quais são as vantagens do polimorfismo em C++?

O polimorfismo em programação, especialmente em linguagens orientadas a objetos como C++, oferece vários benefícios que contribuem para a flexibilidade do código, a capacidade de estender sistemas de classes e facilitar a manutenção do código. Aqui estão alguns desses benefícios:

Flexibilidade do Código: O polimorfismo permite que objetos de diferentes classes sejam tratados de maneira uniforme através de ponteiros ou referências da classe base. Isso proporciona flexibilidade ao código, permitindo que um único trecho de código manipule diferentes tipos de objetos sem a necessidade de conhecer detalhes específicos de cada classe derivada.

Capacidade de Estender o Sistema de Classes: O polimorfismo facilita a extensão do sistema de classes. Ao adicionar novas classes derivadas, é possível estender o comportamento do programa sem modificar o código existente. Isso promove a reutilização do código e facilita a adição de novos recursos ao sistema.

Encapsulamento e Abstração: O uso de polimorfismo contribui para o encapsulamento e a abstração de conceitos. A classe base pode fornecer uma interface abstrata, escondendo os detalhes de implementação das classes derivadas. Isso melhora a modularidade do código, tornando-o mais fácil de entender e manter.

Polimorfismo de Tempo de Execução: O polimorfismo de tempo de execução (runtime polymorphism) é alcançado por meio de funções virtuais. Isso significa que a decisão sobre qual implementação específica chamar é tomada em tempo de execução. Isso permite uma adaptação dinâmica do comportamento do programa, o que é particularmente útil em situações em que as decisões só podem ser tomadas em tempo de execução.

Manutenibilidade do Código: O polimorfismo contribui para a manutenibilidade do código, uma vez que alterações em uma classe derivada podem ser feitas sem impactar o código que interage com a classe base. Isso é valioso ao realizar atualizações ou modificações no sistema, pois reduz o acoplamento entre diferentes partes do código.

Design Orientado a Objetos: O polimorfismo é uma característica fundamental da programação orientada a objetos. Ele promove um design mais modular e organizado, facilitando a modelagem de sistemas complexos por meio de hierarquias de classes e interfaces. Isso é especialmente benéfico ao trabalhar em projetos de grande escala.

Facilita Testes e Simulações: O polimorfismo facilita a criação de testes e simulações. Pode-se usar classes derivadas que implementam interfaces específicas para testar componentes isoladamente, tornando mais fácil a verificação de funcionalidades individuais do sistema.

Reusabilidade do Código: O polimorfismo permite a criação de código mais genérico e reutilizável. Classes base podem ser projetadas para serem aplicáveis a uma variedade de situações, e as classes derivadas podem fornecer implementações específicas conforme necessário. Isso promove a reutilização do código em diferentes contextos.
